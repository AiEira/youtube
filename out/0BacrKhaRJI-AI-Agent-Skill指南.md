## 核心觀點

換更強的 agent 不如配好 skill。作者實測數十個 agent 後的結論：真正決定生產力上限的，不是 agent 本身，而是 skill 配置。

---

## 第一類：元 Skill（地基）

這類 skill 不負責具體任務，而是**擴展 agent 的能力邊界**。第一天就該裝。

### Skill Creator（Anthropic 官方）

把成熟工作流變成可複用 skill。不用研究複雜格式，像給同事交代工作一樣說大白話，一分鐘產出標準 skill。

- 適合：將 SOP 變成 skill
- 建議：全局安裝，跨專案調用

### Find Skills

不是普通搜尋插件——agent 發現自己不會做某任務時，**自動拆解需求、去 skill.sh 全網搜尋**，按安裝量和作者信譽挑選最佳選項，一行命令安裝。

> Skill Creator 讓 agent 自己造工具，Find Skills 讓 agent 去外面找現成的。兩個配合 = 能力邊界無限擴展。

## 第二類：軟體開發（守住工程底線）

核心目標：終結「邏輯閉環但無法落地」的代碼幻覺。

### Superpowers — TDD 正規化

把測試驅動開發變成 agent 的硬規則：

1. 頭腦風暴，磨清需求 → 出設計文檔
2. 大任務拆成小碎活，每件有驗證標準
3. 子 agent 自己跑：寫測試→寫代碼→重構（紅綠重構循環）
4. 兩輪內部審計：一輪對需求、一輪挑質量
5. 測試全過後，合併/留分支/丟掉，選項交給你

雖然單次多花一點時間，但第一遍就寫到 80 分以上，省掉後續反覆 debug。

### JStack（Gary Tan / YC 總裁）

在 agent 裡內置 **23 個專家角色**（CEO、設計師、發布工程師…），斜杠命令直接調用：

| 命令 | 階段 | 作用 |
|------|------|------|
| `office hours` | 動手前 | YC 靈魂拷問：6 個尖銳問題掐死不靠譜假設 |
| `plan CEO review` | 規劃 | 站在 CEO 高度審視計劃 |
| `review` | 代碼複審 | 盯 CI 能過但上線會爆的工程隱患 |
| `QA` | 測試 | 真的打開瀏覽器，像真人測試員點擊驗證 |
| `shape` | 發布 | 自動同步→跑測試→推代碼→開 PR，一氣呵成 |

Gary Tan 2026 年產出是 2013 年的 240 倍——不是代碼行數多，而是一個人頂一支小團隊。

### 前端大神 Mart 的 Skill

TypeScript 佈道者，解決人與 agent「溝通對不上」的問題。核心理念：**寧願前期多花幾分鐘對齊需求，也不要在後期花幾小時處理低質量代碼**。

| 命令 | 作用 |
|------|------|
| `grammy` 系列 | 拷問模式：不馬上動手，反覆追問細節直到需求清晰 |
| `triage` | Issue 分診：輕重緩急排序，不修細枝末節而忽略核心阻塞 |
| `improve` | 架構急救包：每隔幾天跑一次，全局審視，找出會越來越難改的地方 |

## 第三類：前端頁面設計（去 AI 味）

### Front End Design（Anthropic 官方）

核心：洗掉 AI 味。不是套用組件，而是根據產品調性推敲質感紋理和非對稱佈局。字體比例、模塊留白都處理到位——從「能看」變成「耐看」。

### UI UX Pro Max

直接配一個設計總監。後台內置 **160+ 行業深度規則**：
- 金融/醫療介面：什麼配色體現安全感、什麼字體顯專業
- 列出避坑指南：哪些設計在商業場景絕對不能碰
- 生成可持久化複用的設計系統：下次新專案直接丟給 agent

分工：Front End Design 把畫面畫得出彩，UI UX Pro Max 把產品做得專業。

## 第四類：內容創作（寶玉工具箱）

十幾個 skill，打通生產到發布全流程：

| Skill | 特點 |
|-------|------|
| **Cover Image** | 五維控制系統：構圖×色調×渲染風格×文字排版×情緒基調，77 種預設組合 |
| **信息圖** | 21 種專業佈局（魚骨圖、漏斗圖、金字塔圖…），自動讀懂文案邏輯推薦佈局 |
| **小紅書 Image** | 長文→1-10 張卡通輪播卡，11 種視覺風格×8 種排版模式 |
| **Markdown to HTML** | 解決公眾號不支持 Markdown 的痛點；多套主題、自動代碼高亮、外鏈轉文末引用 |
| **翻譯** | 四步流程：分析→翻譯→校正→潤色；可指定讀者群，語氣自動調整 |
| **發布微信/微博** | 一鍵跨平台分發，長文或貼圖模式任選，agent 內直接閉環 |

## 總結

俊旺的最終建議：與其糾結用 Codex 還是 Claude Code，不如根據自己的工作流程和使用場景，打磨真正適合自己的 skill 組合。

> *Skill 沒配對，換再強的 agent 也是浪費時間。*
