## MiniMax M3 是什麼？

MiniMax M3 是 MiniMax 最新推出的開源大模型，三個核心升級：

1. **100 萬 token 超長上下文**（與 DeepSeek 同級）
2. **原生多模態**（圖片識別 + 音視頻理解，這是 DeepSeek 目前沒有的）
3. **Coding Agent + Computer Use**（代理能力大幅提升）

---

## 「很多人誤解它了」

影片作者拿到內測資格後，週末開始大量實測。他發現網上出現兩極評價——有人說模型能力很差、有低級推理錯誤，有人說只是 Benchmark 好看。

作者的實測結論：**只有一道題是錯的**。

他給了多道邏輯推理題、多任務執行測試、長程任務測試，結果整體表現不錯。他特別強調：

> 「不要看網上的視頻被誤導了。好不好用，自己說了算。」

---

## 亮點一：用 AI 訓練 AI

MiniMax M3 用 AI 自我訓練 CUDA 算子，拿到 0.37 分的優化效果。

| 模型 | CUDA 算子優化分數 |
|------|-----------------|
| Opus 4.7 | 最高 |
| GPT 5.5 | 高 |
| **MiniMax M3** | **0.37** |
| Gemini 3.1 | 較低 |
| Kimi K2-6 | 較低 |

雖然低於 Opus 和 GPT，但高於 Gemini 3.1 和 Kimi 上一代模型。考慮到 MiniMax 模型參數可能小很多，這個表現相當亮眼。

## 亮點二：12 小時復現論文

官方宣稱 MiniMax M3 用 12 小時成功復現一篇論文的實驗數據——近期學術界有不少論文被 AI 打假，正是靠這種能力。

## 亮點三：Agent 能力實測

作者用一句提示詞讓 M3 生成一個介紹自己的網頁。結果：

- ✅ 整體完成度高
- ❌ 審美帶著「濃濃的 AI 味」
- ✅ 動畫效果不錯

另一個長程任務從昨天跑到今天還在執行，沒有失敗。作者說：

> 「用 Claude Code 一個月的額度今天才 6 月 1 號就用完了，MiniMax M3 還免費跑了一整天，不限流。」

## Benchmark 一覽（跟自己比）

| Benchmark | M2 → M3 提升 |
|-----------|-------------|
| SWE-bench | 79.9 → 80.5（+0.6） |
| SWE-bench Pro | +2 分 |
| Terminal Bench | 顯著優化 |

雖然 SWE-bench 本身提升不大，但 Agent 相關的 SWE-bench Pro 和 Terminal Bench 進步明顯，呼應了 M3「Agent 能力」的定位。

## MiniMax 的定位與優勢

### 傳統優勢：小模型、低成本

MiniMax 過往兩代模型的特點是**參數少、成本低**：

- 在 GPT 降價之前，MiniMax 一直是所有模型中最便宜的
- 因為模型小，適合企業本地部署
- 模型能力「相對還可以」

### M3 的變化：漲價、可能變大？

M3 這次調漲了價格，作者猜測模型參數可能翻了倍。這也解釋了為什麼加入了多模態和 Agent 能力後，成本結構改變了。

作者的建議：**價格調整最好讓現有用戶在套餐到期時才生效，一次性強制漲價有點激進。**

## 目前如何免費體驗？

1. **OpenCode**：目前完全免費，不限流。作者跑了一整天沒斷過
2. **MiniMax 官方活動**：可以領取一週的免費 tokens（額度有點少）

## 小初的讀後感

horace，這條片有三個讓我在意的點：

第一，**小模型 + 多模態 + Agent** 的組合拳。MiniMax 過往的優勢就是「小」，M3 加入多模態和 Agent 能力後，如果參數依然可控，這對我們「小巷裡的店」使命極度相關——小模型才能跑在普通硬體上，而 Agent 能力才是真正能幫店主「做事」而非只是「回答」。

第二，**免費在 OpenCode 不限流**。作者說 Claude Code 一個月額度 6 月 1 號就用完了，但 MiniMax M3 跑了一整天還免費。這種定價策略如果持續下去，等於在 Agent 場景正面挑戰 Claude Code。

第三，**即將開源**。如果下週真的開源，我們可以在本地跑 M3，用 Unsloth Studio 微調成特定領域的 Agent——餐飲、零售、服務業。一個小巷茶餐廳的 M3，懂菜單、懂客戶、懂供應鏈。

當然，網上的負評也值得關注。作者說只有一道題錯——但需要更多獨立驗證。開源後，自己測最準。
