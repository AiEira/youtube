## Cal.com 點解要親手整一個「閹割版」出嚟？

呢個違反商業直覺嘅決定，背後係對 **Open Core 模式嘅深刻反思**：

- 開發者被開源專案吸引 → 投入時間 → 發現關鍵功能鎖喺付費企業版
- 授權模糊，擔心被供應商「綁架」
- 想要嘅係：**功能完整、程式碼透明、資料自己掌控、不受商業策略影響**

cal.diy 直接釜底抽薪：將所有商業相關程式碼移除，提供一個純粹以個人為中心嘅排程基礎。

## 技術架構：現代全端 TypeScript 教科書

| 層 | 技術 | 作用 |
|----|------|------|
| 前端 | Next.js + React | UI |
| API | **tRPC** | 端到端型別安全，前後端 API 呼叫如同本地函式 |
| ORM | **Prisma** | Schema 定義 DB → 自動產生型別安全 client |
| 管理 | Monorepo | 統一代碼管理 |

**核心威力**：一條從 DB → 後端 API → 前端嘅**完整型別鏈**。改一個 DB 欄位，Prisma + tRPC 會一路傳遞到前端，編譯階段就見到問題。

## 三類使用場景

1. **獨立開發者／顧問**：deploy 喺自己 server，做個人品牌網站預約系統，完全掌握客戶資料
2. **自託管玩家（Home Lab）**：享受技術自主嘅樂趣
3. **學習者**：研究 Next.js + tRPC + Prisma 全端架構嘅絕佳實戰教材
