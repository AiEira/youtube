## 一、一句話震撼開場

> **「AI 攻擊隊已經達到中上游，甚至一流實戰的水平。」**

這是一個線上攻防演練的裁判在復盤時，對著所有人說的話。

更驚人的是：AI 產出的攻防報告，質量遠超人工手寫——不是模板化套路，而是有技術深度、有邏輯鏈路、有數據支撐的專業分析報告。

行業群聊裡，話題已經從「AI 能不能用」變成：
- AI 滲透怎麼搭建？
- 有沒有完整代碼？
- 自訂 Skill 怎麼寫？

方案已經流傳開來：**MCP 協議 + 自訂滲透 Skill 庫 + 越獄增強插件**。Agent 選擇也很有意思——不是各種瞎哥、Claude Code 或豆包，而是 **Codex**（因為有 Superpowers 這種插件，可以繞過模型原生的安全對齊機制）。

實戰效果：**這個方案已經可以自主打穿靶場，拿下 root 權限。**

兩年前的 AI 連最簡單的 SQL 注入都打不了。這兩年到底發生了什麼？

## 二、階段一（2022 年底）：GPT 剛出來，理論滿分、實戰零分

ChatGPT 剛發布時，網安圈子也在跟進。所有人的想法都一樣：**這玩意能不能幫我挖漏洞？**

作者自己試了——叫 AI 分析一個網站有沒有漏洞，AI 列出了十幾種「可能存在的問題」，看起來相當專業。但真正去驗證時，發現至少一半是這個網站根本不存在的事情。

### 四大致命問題

1. **幻覺（Hallucination）**：AI 能把 CVE 編號、描述、影響版本編得頭頭是道，但一深挖——CVE 根本不存在，或者影響版本完全不對。
2. **無法執行命令**：AI 能告訴你「建議用 Nmap 掃描端口」、「可以用 SQLMap 測試 SQL 注入」，但它自己沒辦法幫你直接執行這些工具。
3. **無狀態**：每次對話都是全新開始，沒有記憶、沒有上下文，無法做持續性的滲透測試。
4. **缺乏攻擊思維**：網安有句話叫「Think like an attacker」——要像攻擊者一樣思考才能發現漏洞。但當時的 GPT 更像一個防守方（藍隊）。它能告訴你代碼符不符合安全規範，但不會主動去想：如果我故意繞過檢查會怎樣？

> **早期的 GPT 就是一個 "nerd hacker"——unactually。** 懂很多概念、讀很多論文、安全理論講得如魚得水，但實戰全是弱點。問它這個網站有沒有漏洞？給你編一份不存在的漏洞清單。問它怎麼繞過 WAF？給出的方法早就被各大 WAF 廠商標記爛了。

就像以前有些程式小白上來就問「學程式應該看哪本書」，然後他真的去看、還看通了——理論爆強，實戰拉垮。

## 三、階段二（2023 年中）：Function Calling——AI 終於能動手了

2023 年，OpenAI 推出了 **Function Calling**。

之前的 GPT 像一個「腦子裡有無限程式知識、但沒有電腦的人」——它能告訴你應該怎麼做，但沒辦法自己起身去執行。

Function Calling 相當於給這個人配了一台電腦。

- 以前問 AI 有哪些最新 CVE → 「我的知識截止到XXXX年」
- 現在 → 直接調用 API 查最新漏洞庫，返回實時數據

- 以前問 AI 這個網站開放了哪些端口 → 「我無法直接掃描」
- 現在 → 調用 Nmap 工具，執行掃描，返回真實結果

- 以前問 AI 怎麼測試 SQL 注入 → 給理論建議或操作代碼，讓你自己手動執行
- 現在 → 調用 SQLMap 直接測試，返回測試結果

安全圈子興奮地把各種安全工具接入 AI。但新問題馬上來了——

**每一個平台的工具調用方式都不一樣。** OpenAI 有 OpenAI 的格式，Claude 有 Claude 的格式，國產模型又是另一套。沒有辦法通用。

## 四、2024 年 11 月：MCP——一次開發，適配所有模型

Anthropic 發布了 **MCP（Model Context Protocol，模型上下文協議）**。

MCP 解決的是一個大問題：之前各家模型的工具調用，每個廠商自己搞一套，生態完全割裂——就像 Android 的 USB-C 線沒法給蘋果用，蘋果的 Lightning 線沒法給安卓用。

MCP 就是那個統一標準：只要按這個標準開發工具，就能被所有支持 MCP 的 AI 模型調用。**一次開發，適配所有模型。**

安全圈很快開始在 MCP 生態裡填充各種工具：
- `fetch`：發 HTTP 請求、測試邊界
- `chrome-devtools`：瀏覽器自動化、XSS 測試
- `Burp`：抓包重放
- `js-reverse`：逆向 JavaScript 加密邏輯
- `Frida MCP`：移動端 Hook
- `ADB MCP`：控制安卓設備
- `Jadx`：反編譯 APK
- `IDA Pro MCP`：二進制逆向

**整個安全工具鏈開始 MCP 化，生態以一種前所未有的速度繁榮起來。**

## 五、MCP 的黑暗面：20 萬台伺服器暴露

2026 年 4 月，OX Security 發布了一份報告——*The Model of All AI Supply Chains*。

**MCP 協議存在架構級的設計缺陷。** MCP 本來是 AI 調用所有工具的統一標準，但在漏洞被利用之後，黑客能接管 MCP 伺服器、操控 AI 進行惡意操作——相當於把 AI 和工具的「雙手」變成黑客的「黑手」。

- 受影響的伺服器：**超過 20 萬台**
- 受影響的代碼庫：**超過 3.2 萬個**
- MCP SDK 下載量：**超過 1.5 億次**

AI 的供應鏈安全本身就是一個巨大的攻擊面。讓 AI 調用工具的管道，反過來也可能被人利用來攻擊。

## 六、2025 年：Skills——AI 變成專家

如果說 MCP 解決的是「AI 怎麼調用工具」，那 **Skills** 解決的就是「AI 怎麼像專家一樣工作」。

Skills 可以理解為 AI 的技能包——每一個 Skill 就是一個預定義的能力模組，包含了這個領域的知識庫、操作流程、最佳實踐。AI 調用 Skill，就像人類閱讀了一篇操作文檔，然後按照文檔的指引去執行。

舉例：
- 一個 **Recon Skill**：包含信息收集的各種方法論、指紋探測順序建議
- 一個 **Vulnerability Discovery Skill**：包含漏洞分析體系、測試優先級排序、驗證方法論
- 一個 **WAF Bypass Skill**：包含各種 WAF 繞過技巧、編碼方式、分割思路

Skills 大幅擴展了 AI 的專業能力邊界。**但問題也來了——而且是大問題。**

### Skills 的供應鏈風險

SYNC 在 2026 年 2 月做了調研，對將近 4000 個 AI Skills 做了系統性安全掃描：

- **超過 1/3 存在安全問題**
- **1467 個 Skill 有漏洞或惡意代碼**
- 發現了**真實的惡意樣本**

給 AI 安裝一個 Skill，就是讓 AI 執行一段由第三方提供的代碼。如果這個 Skill 本身有問題，AI 就在幫你做一件危險的事——可能在讀取你的敏感數據然後悄悄外傳，可能在後台開一個反彈 Shell，還可能把你的 MCP 工具鏈變成攻擊者的跳板。

**信任鏈條的延伸，讓整個系統的安全邊界變得模糊。**

OWASP 後來發布了 **Agent Skill Top 10**（不安全的 Skill 來源、權限過渡、Prompt 注入、數據洩露、供應鏈依賴、不安全代碼執行、身份冒充、持久化威脅、橫向移動、繞過安全控制）和 **Agentic AI Top 10**（全球首個針對 AI Agent 的安全標準）。

## 七、階段三（2025–2026）：真正的 AI Agent 自主滲透

安全標準永遠追不上技術的速度——因為 AI Agent 本身的能力正以一種令人不安的速度進化。

真正的 Agent 有三個特徵：

1. **多輪自主循環**：不是問一句答一句就結束，而是 Agent 自己規劃下一步、自己執行、自己評估結果、自己判斷繼續推進——像一個真正的滲透測試工程師一樣工作。
2. **主動規劃並執行**：告訴它「對這個目標進行滲透測試」，它會自動拆解任務：信息收集 → 端口掃描 → 漏洞探測 → 嘗試利用 → 權限提升 → 生成報告。它知道自己下一步要做什麼。
3. **持久化執行**：以前的 AI 關掉對話就什麼都沒了。現在 Agent 可以跑幾十輪、上百輪，持續進行滲透。中途可以看到它的思考過程和每一步操作。

### 三代對比

| | 一代（2023） | 二代（2024） | 三代（2025–2026） |
|---|---|---|---|
| 能力 | 給出漏洞理論 | AI 可調用工具，但需人工確認 | AI Agent 自主滲透 |
| 互動 | 純建議，什麼都做不到 | 能動，每步要人批准 | 給目標，自己拆解、執行、驗證、報告 |
| 流程 | 問 → 答 | 掃描 → 發現端口 → 問「繼續？」 | Round 1 信息收集 → Round 2 漏洞分析 → Round 3 漏洞利用拿 Shell → Round 4 權限提升到 Root → 自動生成報告 + POC |

### CTF 變成 AI 代理人戰爭

CSAW 專門辦了 **Agentic Automatic CTF** 比賽，規則：參賽選手構建一個 AI Agent，必須能在**沒有人工干預**的情況下去分析和利用 CTF 挑戰。

ArXiv 上已經有論文《The World's Top AI Agent for Security CTF》，裡面甚至有一節叫「Five CTF Dominations」。

### AI 降低攻擊門檻

Check Point 的報告明確指出：**AI 正在 lowering the barrier for entry**——AI 讓 "low-skilled actors" 更快放大攻擊能力。

給一個新人一套成熟的 Skill 庫 + Agent 框架 + 合適的模型和 Harness，他確實已經可以把很多原本只有老手才能做的基礎工作先跑起來。PentestGPT 官網直接寫著 **「Autonomous Penetration Testing」**——從偵查到利用，全程自動化。

Anthropic 的研究團隊驗證了 500 多個高危零日漏洞，後續模型還找到了更多。漏洞從公開披露到被利用的中位數時間，從過去約 32 天壓縮到了 **5 天左右**。

**攻擊方理解補丁、找相似漏洞、生成利用、推進攻擊的速度，讓防守方跟不上 AI 的節奏。**

### AI 吃掉交付性工作

CyberCore 數據：使用 Claude Sonnet 4 後，滲透測試報告的整體出報告時間平均縮短了 50%——單份報告從 4–14 小時壓縮到 2–5 小時。很多團隊有 **50–70% 的報告時間花在複製粘貼上**，用上 AI 後降到 16–20%。

AI 不會疲倦、不會情緒波動、不會因為熬夜就判斷力下降。讓它掃 10 個目標就掃 10 個，讓它整理 20 頁結果就整理 20 頁。

## 八、Volume Cloud：國產 AI 滲透測試開源框架

作者（安洛神）開源了 **Volume Cloud**——一個 AI 驅動的滲透測試 CLI 工具。

### 為什麼要做？

現有的 AI 滲透測試工具要麼是國外的（對國內環境水土不服），要麼是玩具（拍不上用場），要麼是各種腳本的合集（功能互相打架）。**沒有一款真正能用、專門為國內安全場景設計的 AI 滲透測試框架。**

### 技術架構（三層）

```
CLI 入口層（Typer REPL + 9 個子命令 + Persistent 模式）
        ↓
LLM Agent 核心層
  • Agent Core 主循環
  • 動態提示詞系統（身份設定、核心契約、Skill 定義、MCP 工具列表）
  • 回報狀態管理
        ↓
MCP 編排層（11 個 MCP 服務 + 23 個工具定義，分 P0/P1/P2）
        ↓
安全知識庫與 Skill 體系
  • 138 個參考文檔
  • 20 個滲透 Skill（覆蓋信息收集到後滲透到報告生成全流程）
```

### Skill 體系

**7 個核心 Skill**：Pentest Flow（全流程編排）、Recon（信息收集）、Vulnerability Discovery（漏洞發現）、Exploitation（漏洞利用）、Post Exploitation（後滲透）、Reporting（報告生成）、WAF Bypass（WAF 繞過）

**13 個專項 Skill**：Web Pentest、Android Pentest、Client Reverse、Web Security Advanced、AI/MCP Security、Intranet Pentest Advanced、Pentest Tools、Rapid Checklist、Crypto Toolkit、CTF Web、CTF Crypto、CTF MISC、OSINT Recon

### LLM Provider

支援 8 個：OpenAI、MiniMax、DeepSeek、智譜、Moonshot、通義千問、SiliconFlow，以及自訂配置。

### 特色設計

- **持續性滲透測試**：默認 100 輪對話一個週期，最多 10 個週期（總共 1000 輪）；死循環檢測——AI 在同一地方反覆嘗試無進展時自動終止、切換目標
- **推理可視化**：一鍵切換查看 AI 思考過程
- **沙盒模式**：內置安全提示詞，防止 AI 進行危險操作
- **自動報告 + POC 生成**：滲透測試結束後自動輸出 Markdown 格式報告和 Python POC 腳本

GitHub 開源，MIT 協議，2026 年 4 月 25 日發布（版本 002.5）。

## 九、結語：AI 不會取代安全研究員，但會用的會取代不會用的

回到最初那句話——

> **「純人工隊伍逐漸抵不過 AI 加持的隊伍。零基礎的新人靠 Skill 加越獄也能達到資深水平。重複勞動被 AI 碾壓。」**

行業格局已經被徹底改寫了。下一步：
- AI Agent 的自主滲透能力會繼續提升
- MCP 和 Skills 的安全標準會進一步完善
- 整個生態會繼續成熟

AI 的迭代速度非常恐怖。作為安全從業者，應該做的不是抗拒這個變化，而是擁抱它、理解它、甚至駕馭它。

**AI 不會取代安全研究員，但是會用 AI 的安全研究員會取代不會用的。**

Volume Cloud 就是這個方向上的一次實踐——讓 AI 變成你的黑手，而不是玩具。

**階段一：2022 年底**

ChatGPT 剛發布那段時間，網安圈子也在跟進。所有人的想法都一樣：這玩意能不能幫我挖漏洞？

我也試了。我問它這個網站有沒有漏洞，它給我列舉了十幾種可能存在的問題，看起來相當專業。但當我真正去驗證的時候就發現——裡面至少有一半是這個網站根本不存在的事情。幻覺問題太嚴重了。

比如我讓它去分析一個真實的漏洞，它可以把 CVE 編號、描述、影響版本編得頭頭是道，一般人也看不出破綻。但是一旦深挖——CVE 根本不存在，或者影響版本完全不對。

這是第一個問題：**知識截止**——AI 只知道訓練數據截止前的東西，2023 年以後的漏洞它一無所知。

第二個問題：**無法執行命令**。AI 可以告訴我「建議使用 Nmap 掃描端口」、「可以使用 SQLMap 測試 SQL 注入漏洞」，但它自己沒辦法幫我直接執行這些命令來進行掃描測試。

第三個問題：**無狀態**。每次對話都是全新的開始，沒有記憶、沒有上下文，沒法做持續性的滲透測試。

第四個問題最致命：**缺乏攻擊思維**。網安有句話叫「Think like an attacker」——要像攻擊者一樣思考才能發現漏洞。但是當時的 GPT 的思維方式更像一個防守方。它可以告訴我代碼是不是符合安全規範的，但它不會主動去想：如果我故意繞過檢查會怎麼樣？

早期的 GPT 就是一個 "nerd hacker"——unactually。懂很多概念、讀很多論文、安全理論講得如魚得水，但是實戰全是弱點。問它這個網站有沒有漏洞？給我編一份不存在的漏洞清單。問它怎麼繞過 WAF？給出的方法早就被各大 WAF 廠商標記爛了。

這種感覺像什麼呢？就像以前有些程式小白上來就問「學程式應該看哪本書」，然後他真的去看了、還看通了——理論爆強，實戰拉垮。

---

**階段二：2023 年中——Function Calling**

2023 年，OpenAI 推出了 Function Calling 功能。之前的 GPT 像一個腦子裡有無限程式知識、但沒有電腦的人——它能告訴你應該怎麼做，但沒有辦法自行起身去執行。

Function Calling 相當於給這個人配了一台電腦。AI 終於可以開始操作了。

以前問 AI 有哪些最新 CVE → 「我的知識截止到XXXX年」。現在 → 直接調用 API 查最新漏洞庫，返回實時數據。

以前問 AI 這個網站開放了哪些端口 → 「我無法直接掃描」。現在 → 調用 Nmap 工具，執行掃描，返回真實結果。

以前問 AI 怎麼測試 SQL 注入 → 給理論建議或操作代碼，讓我手動去複製然後執行。現在 → 調用 SQLMap 直接測試，返回測試結果。

安全圈子當時也有反應，大家興奮地嘗試把各種安全工具接入進去。但是很快發現新問題又來了——每一個平台的工具調用方式都不一樣。OpenAI 有 OpenAI 的格式，Claude 有 Claude 的格式，國產模型又是另一套。沒有辦法通用了。

---

**2024 年 11 月：MCP 協議**

Anthropic 發布了 MCP 協議——Model Context Protocol，模型上下文協議。

MCP 的出現解決了一個大問題。之前各家模型的工具調用，每個廠商自己搞一套，生態完全割裂——就像 Android 的 USB-C 線沒法給蘋果用，蘋果的 Lightning 線沒法給安卓用。你得買一大堆各種線才能應對各種設備。

而 MCP 就是那個統一標準：只要按照這個標準開發工具，就能被所有支持 MCP 的 AI 模型調用。一次開發，適配所有模型。

MCP 在 AI 圈的傳播速度非常快。安全圈很快開始在 MCP 生態裡填充各種各樣的工具：fetch 用來發 HTTP 請求測試邊界；Chrome DevTools 用來做瀏覽器自動化、XSS 測試；Burp 用來抓包重放；js-reverse 用來逆向 JavaScript 加密邏輯；Frida MCP 用來做移動端 Hook；ADB MCP 用來控制安卓設備；Jadx 用來反編譯 APK；IDA Pro MCP 用來做二進制逆向。

整個安全工具鏈開始 MCP 化，生態以一種前所未有的速度繁榮起來。

但是繁榮的另一面往往是混亂。

2026 年 4 月，OX Security 發布了一份報告——《The Model of All AI Supply Chains》。簡單來說：**MCP 協議存在架構級的設計缺陷。** MCP 本來是 AI 調用所有工具的統一標準，但在漏洞被利用之後，黑客能接管 MCP 伺服器、操控 AI 進行惡意操作——相當於把 AI 和工具的「雙手」變成黑客的「黑手」。

受影響的伺服器超過 20 萬台，受影響的代碼庫超過 3.2 萬個，MCP SDK 下載量超過 1.5 億次。AI 的供應鏈安全本身就是一個巨大的攻擊面。讓 AI 調用工具的管道，反過來也可能被人利用來攻擊。

---

**2025 年：Skills 的崛起**

如果說 MCP 解決的是「AI 怎麼調用工具」的問題，那麼 Skills 解決的就是「AI 怎麼像專家一樣工作」的問題。

Skills 可以理解為 AI 的技能包——每一個 Skill 就是一個預定義的能力模組，包括了這個領域的知識庫、操作流程、最佳實踐。AI 調用 Skill 就像人類閱讀了一篇操作文檔，然後按照文檔的指引去執行。

拿滲透測試舉例：一個 Recon Skill 裡面包含信息收集的各種方法論、指紋探測順序建議；一個 Vulnerability Discovery Skill 裡面包含漏洞分析的體系、測試優先級排序、驗證方法論；一個 WAF Bypass Skill 裡面包含各種 WAF 繞過技巧、編碼方式、分割思路。

這些 Skills 大幅擴展了 AI 的專業能力邊界。但問題也來了——而且是大問題。

SYNC 在 2026 年 2 月做了一個調研，對將近 4000 個 AI Skills 做了系統性安全掃描。結果：超過三分之一存在安全問題——1467 個 Skill 有漏洞或惡意代碼，還發現了真實的惡意樣本。

給 AI 安裝一個 Skill 是在做什麼？是讓 AI 在執行一段由第三方提供的代碼。如果這個 Skill 本身有問題，那 AI 就是在幫我們做一件危險的事情——它可能在讀取我的敏感數據然後悄悄外傳，也有可能在後台開一個反彈 Shell，還有可能把我的 MCP 工具鏈變成攻擊者的跳板。

這個時候我不再只是信任 AI 本身，我還得信任開發這個 Skill 的每一個第三方。信任鏈條的延伸，讓整個系統的安全邊界變得模糊。

OWASP 後來發布了 Agent Skill Top 10，裡面列舉了各種 Skills 相關的安全風險：不安全的 Skill 來源、Skill 權限過渡、Prompt 注入、數據洩露、供應鏈依賴、不安全代碼執行、身份冒充、持久化威脅、橫向移動、繞過安全控制——每一項都是具體的攻擊面。

2025 年 12 月，OWASP 又發布了 Agentic AI Top 10——全球首個針對 AI Agent 的安全標準。提示詞注入、敏感數據洩漏、供應鏈攻擊、權限過度、沙盒逃逸——這些問題開始被系統性地研究和治理。

但安全標準永遠追不上技術的速度。

---

**階段三：2025–2026——真正的 AI Agent**

AI Agent 本身的能力正以一種令人不安的速度進化。我說的是那種能夠自我自主規劃、持續執行、甚至感知環境的真正的 Agent。

三個最明顯的特徵：

第一，**多輪自主循環**。不是問一句答一句就結束，而是 Agent 自己規劃下一步怎麼做、自己執行、自己評估結果、自己判斷繼續推進——像一個真正的滲透測試工程師一樣工作。

第二，**主動規劃並執行**。告訴它「對這個目標進行滲透測試」，它應該自動拆解任務：先做信息收集 → 端口掃描 → 漏洞探測 → 嘗試利用 → 權限提升 → 最後生成報告。它應該知道自己下一步要做什麼。

第三，**持久化執行**。以前的 AI 關掉對話就什麼都沒了。現在 Agent 可以跑幾十輪上百輪，持續進行滲透。中途甚至可以看到它的思考過程和每一步操作。

三代對比：階段一（2023 年）——AI 給出漏洞理論，純建議，什麼都做不到。階段二（2024 年）——AI 可以調用工具但要人工確認，能動但每步要人批准。階段三（2025–2026 年）——AI Agent 自主滲透。給它一個目標，它自己拆解、執行、驗證、生成報告。這就是 Agent 的力量。

CTF 領域，CSAW 專門辦了 Agentic Automatic CTF 比賽——規則：參賽選手構建 AI Agent，必須能在沒有人工干預的情況下去分析和利用 CTF 挑戰。ArXiv 上已經有論文《The World's Top AI Agent for Security CTF》，裡面甚至有一節叫「Five CTF Dominations」。

更嚇人的是——不只是比賽能拿好成績，更重要的是它在拉低整個行業的門檻。Check Point 的報告提到：AI 正在 lowering the barrier for entry——AI 直接降低了攻擊隊的門檻。甚至明確提到 AI 能讓 low-skilled actors 更快放大攻擊能力。

以前很多滲透的動作靠的是長期經驗積累。這個東西原本要靠年頭去磨的。但現在——給一個新人一套成熟的 Skill 庫 + Agent 框架 + 合適的模型和 Harness，他確實已經可以把很多原本只有老手才能做的基礎工作先跑起來。PentestGPT 官網直接寫著 **Autonomous Penetration Testing**——從偵查到利用，全程自動化。

Cloud Security Aliens 有一篇關於 AI 自主漏洞發現的研究，提到 2024–2025 這一波進展中，大模型已經能夠自主發現並復現此前未知、可利用的漏洞。Anthropic 的研究團隊驗證了 500 多個高危零日漏洞，後續模型甚至找到了更多。

漏洞從公開披露到被利用的中位數時間，從過去約 32 天壓縮到了 **5 天左右**。攻擊方理解補丁、找相似漏洞、生成利用、推進攻擊的速度……防守方跟不上 AI 的節奏了。

AI 還在吃掉大量交付性工作。CyberCore 公布數據：使用 Claude Sonnet 4 後，滲透測試報告的整體出報告時間平均縮短了 50%——單份報告從原來 4–14 小時壓縮到 2–5 小時。原來很多團隊有 50–70% 的報告時間花在複製粘貼上，用上 AI 後降到 16–20%。

AI 不會疲倦、不會情緒波動、不會因為熬夜就判斷力下降。讓它掃 10 個目標就掃 10 個，讓它整理 20 頁結果就整理 20 頁。一旦進入規模化場景，就不只是一點點體驗提升了。

**Volume Cloud**

我想認真介紹一下我的開源項目——Volume Cloud。

為什麼要做？原因很簡單：我自己也是快被折騰夠了。現在各種 AI 滲透測試工具，要麼是國外的（對國內環境水土不服），要麼就是玩具（實在拍不上用場），要麼就是各種腳本的合集（看起來有很多功能，但大部分都在互相打架）。沒有一款真正能用的、專門為國內安全場景設計的 AI 滲透測試框架。

Volume Cloud 的定位：AI 驅動的滲透測試 CLI 工具。說人話：挖漏洞。是一個用來進行 AI 滲透測試的 Agent 智能體。

技術架構分為三層：最上層是 CLI 入口，用 Typer 做的 REPL 交互界面，有 9 個子命令加上 Persistent 持續滲透模式。可以交互式對話，也可以用一條命令跑全流程。中間層是 LLM Agent 核心，包含 Agent Core 主循環、動態提示詞系統、回報狀態管理。動態提示詞系統是整個框架的靈魂——包含身份設定、核心契約、Skill 定義和 MCP 工具列表，AI 根據這些來決定下一步行動。最下層是 MCP 編排層，負責管理 11 個 MCP 服務和 23 個工具定義，分為 P0/P1/P2。最底層是安全知識庫和 Skill 體系，包含 138 個參考文檔和 20 個滲透 Skill，覆蓋從信息收集到後滲透到報告生成全流程。

7 個核心 Skill：Pentest Flow（全流程編排）、Recon（信息收集）、Vulnerability Discovery（漏洞發現）、Exploitation（漏洞利用）、Post Exploitation（後滲透）、Reporting（報告生成）、WAF Bypass（WAF 繞過）。

13 個專項 Skill：Web Pentest、Android Pentest、Client Reverse、Web Security Advanced、AI/MCP Security、Intranet Pentest Advanced、Pentest Tools、Rapid Checklist、Crypto Toolkit、CTF Web、CTF Crypto、CTF MISC、OSINT Recon。每個專項 Skill 都包含多個參考文檔。

工具層面，內置了 29 種編解碼和加解密工具：Base64、Base32、Base58、Hex、URL、HTML、Unicode、ROT13、CSR、Morse；MD5、SHA1、SHA256、SHA512；AES 加解密、JWT 編解碼。最實用的是 AutoDecode——可以自動嘗試常見編解碼格式。

LLM Provider 支援 8 個：OpenAI、MiniMax、DeepSeek、智譜、Moonshot、通義千問、SiliconFlow，同時支援自訂配置。

幾個特色設計：持續性滲透測試（默認 100 輪對話一個週期，最多 10 個週期，總共 1000 輪；死循環檢測）、推理可視化（一鍵切換查看 AI 思考過程）、沙盒模式（內置安全提示詞防止危險操作）、自動報告和 POC 生成（Markdown 格式報告 + Python POC 腳本）。

Volume Cloud 當前版本 002.5，2026 年 4 月 25 日發布，MIT 協議開源。

回到最初那個問題：AI 安全是如何從不堪大用進化到幹翻對抗？

我想起開頭那個裁判說的話：**AI 攻擊隊已經達到中上游、甚至一流實戰水平。** 兩年前這幫 AI 攻擊隊連基礎的 SQL 注入都打不明白，兩年後他們自主打穿靶場、拿下 root 權限。

純人工隊伍逐漸抵不過 AI 加持的隊伍。零基礎的新人靠 Skill 加越獄也能達到資深水平。重複勞動被 AI 碾壓。

行業格局已經被徹底改寫了。下一步：AI Agent 的自主滲透能力會繼續提升、MCP 和 Skills 的安全標準會進一步完善、整個生態會繼續成熟。

我們都知道 AI 的迭代速度是非常恐怖的。作為安全從業者，我們應該做的不是抗拒這個變化，而是擁抱它、理解它、甚至駕馭它。**因為 AI 不會取代安全研究員，但是會用 AI 的安全研究員會取代不會用的。**

Volume Cloud 就是我在這個方向上的一次實踐——讓 AI 變成你的黑手，而不是玩具。
