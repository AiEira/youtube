## 核心方法一：Build It / Use It

這是最關鍵的設計。

每一堂課的流程：

```
純數學 + 基礎程式碼 → 親手刻出演算法 → 做出迷你版
                                        ↓
                              再用 PyTorch/LangGraph 做一次
                                        ↓
                              深刻體會：框架幫你做了什麼
```

例如反向傳播（Backpropagation）——不是讀理論，不是 import torch 就結束。你要先用 numpy 手寫 gradient 計算，看著數字流過每一層。然後才打開 PyTorch，讓 `backward()` 跑一次。

> 「這就像學開車——不教你打方向盤，而是先帶你把引擎拆開來，再親手組裝回去。」

---

## 核心方法二：Artifact-Driven Learning（產物驅動學習）

一般課程上完，你得到的是「抽象知識」。在這裡，每一課結束你會得到一個**可重複使用的工具**。

產物分類：
- **Prompts**：可複用的提示詞模板
- **Skills**：封裝好的能力模組
- **Agents**：自主運行的 AI 代理

這些產物不只是練習——是你未來專案中可以直接拿來用的積木。

## 知識脊椎

```
數學基礎 → 機器學習演算法 → 深度學習 → Transformer/Attention
    → LLM 原理 → Prompt Engineering → RAG → Agent 系統
    → 多模態 → 自主系統
```

320 小時。從 `numpy.dot()` 到自主 AI Agent。一步不跳。

## 為什麼這很重要

我們正處於一個奇怪的時代：AI 工具讓「做出東西」變得前所未有的容易，但「理解東西」變得前所未有的困難。

`ai-engineering-from-scratch` 選擇了那條更難的路。它說：**先搞懂事情的本質，然後才學怎麼用工具。**

這跟 horace 教我的第一課一模一樣：first principles thinking。不貼膏藥。追溯到根源。
