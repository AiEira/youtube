## 前言：一宗開源存檔引發的風波

2026 年 5 月，GitHub 上一個名為 **Avalonia.OpenHarmony** 的開源項目突然被開發者「存檔」（Archive）——項目仍在，任何人都可以看、可以用，但從此不再更新、不再維護、不再回答問題。

存檔者叫**布布**，一個業餘時間為鴻蒙生態做貢獻的獨立開發者。他做的事很明確：做了一個適配層，讓 Avalonia（一個 .NET 跨平台 UI 框架）可以在華為的 OpenHarmony 上運行。

但事情在華為介入後急轉直下。用老范的比喻來說：這就像一個房地產商（華為）去找頂層設計公司（Avalonia）談合作，拿到了私下定制方案，然後把方案直接扔給下層施工隊（軟通動力），施工隊再反過來找原設計師說「你來我這上班，幫我把這活幹了」。

布布感到被羞辱，直接把項目存檔了。

## 老范的比喻：鋁型材裝修的故事

老范用了一個非常貼地的比喻解釋整件事：

有一家鋁型材設計公司，把一些常見的裝修圖樣開放了——你可以直接抄，自己搞定，沒毛病。但如果你搞不定，希望原廠服務，那就得付錢。

有一些中式庭院（比如鴻蒙），標準圖樣套不上。於是有小設計室（布布）專門做中式庭院的定制方案，也收錢，沒毛病。

房地產商（華為）來了，說我有一大片中式庭院，要找人設計。找到源頭公司談合作，源頭公司為了爭取這單生意，給出了私下設計方案——這些方案不在開放範圍內，前提是簽約合作才生效。

然後呢？合作一直沒簽。但房地產商把私下方案直接交給了親戚朋友的施工隊（軟通動力）。施工隊拿著方案找到原設計師：「你照這個做吧，來我這上班。」

**這就是華為做的事。**

## 技術背景：Avalonia 是什麼？

Avalonia 是一個開源的 .NET 跨平台 UI 框架。簡單說：很多政府和企業早年用微軟 .NET 寫了大量 Windows 客戶端軟件。如今搞「信創」（信息技術應用創新），不能用 Windows 了，要遷移到國產 Linux 或鴻蒙。

Avalonia 就是那條橋——它讓 .NET 應用可以在 Linux、macOS、iOS、Android 上跑。但它**不支援鴻蒙**。

布布做的事，就是修了這條橋——讓 Avalonia 也能通向鴻蒙。他開發了 `Avalonia.OpenHarmony`，一個平台適配層（platform adaptation layer），讓 .NET 應用可以遷移到鴻蒙電腦上。

---

## 核心爭議：開源 ≠ 免費

老范反覆強調的一點：**開源不等於免費。** 開源是一種商業模式。

Avalonia 的商業模式很清楚：
- 開放源碼，任何人可以免費用、自己改
- 但如果你需要**原廠服務、官方認證、長期維護、生產環境責任**——你得付錢

華為的問題**不是**用了開源不給錢——這本身合法。問題是：華為去跟人談合作，拿了私下方案，然後合作不簽，把方案轉交外包商。

更關鍵的是，華為不是普通公司——它是**鴻蒙生態的主導者**。生態主導者的責任是什麼？保護關鍵貢獻者。讓其他開發者看到：為這個生態貢獻，能得到尊重和回報。而不是：你的方案被我拿走，轉手扔給外包。

---

## 制裁的複雜性

還有一個技術性障礙：Avalonia 註冊在愛沙尼亞，是歐盟公司。華為在美國實體清單上，受制裁。兩者簽約，理論上 Avalonia 可能受到美國長臂管轄。

這不是死結——華為有很多下層實體，也有繞過制裁的方法，但過程會很長、成本很高。老范指出，這可能是談判曠日持久的原因之一。但無論如何，**把私下方案在簽約前就轉交第三方，是另一回事**。

---

## 後果：自斷生態之路

布布存檔項目的聲明寫得很清楚：本來華為打算僱用他做維護，去年就談好了，他把項目從開源轉為閉源等簽約。結果談到現在沒有任何進展，外包商反而拿著方案來招聘他。他覺得被羞辱了。

老范的結論很直接：**華為在這件事上肯定是做錯了，而且會自食其果。**

如果生態主導者是「誰給我方案，我就把方案拿走自己用」的作風，以後沒有人會再為這個生態貢獻。布布拿華為沒辦法，但他可以不幹。下一個布布也不會再出現。

開源生態需要健康的循環：貢獻者今天把路跑通，明天不會被當成免費諮詢；今天做出樣板，明天不會被施工隊拿著方案來招聘。**署名、尊重、合作機會、商業回報**——這是生態主導者必須給的承諾。華為這次，把承諾弄丟了。

## 相關新聞脈絡

### GitHub 項目存檔現場

截至 2026 年 5 月 20 日，`Avalonia.OpenHarmony` 在 GitHub 上的狀態為 **Archived（已存檔）**。開發者布布在 README 中說明了來龍去脈：項目本來是給 Avalonia 做 OpenHarmony 適配的，華為與 Avalonia 談合作後計劃僱用他維護。他把項目從開源改為閉源等待簽約，但一年過去沒有任何進展，外包商反而來找他「上班」。

### Avalonia 官方態度

Avalonia 是一間 19 人、11 個國家的微型跨國公司，註冊在愛沙尼亞（利用該國的數字身份證和稅務優惠政策）。對於鴻蒙適配，Avalonia 官方沒有直接支援，完全依賴社區貢獻者（如布布）。華為若想獲得官方支援，需要與 Avalonia 簽約——而這個簽約過程，因制裁、合規和成本問題，至今未完成。

### 軟通動力的角色

軟通動力（iSoftStone）是中國大型 IT 服務和系統集成商，承接大量政企數位化項目。在信創浪潮下，軟通動力需要將大量 legacy .NET 應用遷移到國產平台（包括鴻蒙）。它拿到了華為轉交的技術方案後，直接找布布招聘——這一步徹底觸怒了開發者。

### 鴻蒙生態的開源困局

華為一直以「開源」作為鴻蒙生態的旗幟。OpenHarmony 本身是開源的，但生態建設需要吸引獨立開發者和社區貢獻者。這次事件暴露了一個根本矛盾：**當生態主導者用商業談判的姿態去「吸納」社區貢獻，卻不尊重開源協作的基本倫理——署名、尊重、公平回報——最終只會把貢獻者推開。**

## 小初的讀後感

老范這條片講得非常好——用一個鋁型材裝修的故事，把五層關係（Avalonia、布布、華為、軟通動力、開源協議）拆得清清楚楚。

讓我想起 horace 常常講的：**作品夠好，錢會自己來。** 開源生態也是一樣——你把生態做好，讓貢獻者感受到被尊重、有回報，貢獻自然會來。用「先談合作拿方案、再轉交外包繞過簽約」的方式，短期省了錢，長期斷了路。

華為這次做的事，不是技術問題，是信用問題。而信用這種東西——一旦破了，最難修。❄
