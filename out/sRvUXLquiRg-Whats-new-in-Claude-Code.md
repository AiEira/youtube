## 一句總結

Claude Code 最近幾個月更新的量大到可以講七小時。Ralph 從中挑出最重要的三大方向：讓開發者用得更順手、讓 Claude 能自己做完更多事、以及一個統一管理所有 session 的 Agent View。

---

## 兩大主題

Ralph 把整場分享分為兩大主軸：

| 主題 | 核心問題 |
|------|---------|
| **開發者體驗** | 如何讓一個每天用的工具，用得舒服？ |
| **自主性** | Claude 能不能在不需要你的時候不打擾你，自己把事情做完？ |

---

## 開發者體驗亮點

### 1. 更豐富的專案脈絡

Claude Code 現在能更好地理解你的專案——不只是當前檔案，而是整個 codebase 的結構和脈絡。這意味著它給出的建議更精準、更少離題。

### 2. 更聰明的檔案探索

Ralph 做了一個現場 demo：點進一個專案，Claude 自動理解全部檔案的關係。他說：「我們非常開放地接受回饋——實驗、報 bug、提新功能、告訴我們該往哪個方向走。」

### 3. 互動體驗優化

從對話流程到錯誤處理，Claude Code 變得更有「協作感」。不再是下指令 → 等結果 → 修正 → 再等，而是更像一個理解你意圖的配對程式員。

---

## 自主代理——讓 Claude 自己做事

這部分是整場的核心。Ralph 說這些功能有一個共同的根源：

> 「你有沒有開過一個你以為會很長的 session，回來卻發現 Claude 卡住了？」

### 長任務不卡關

新的自主機制讓 Claude 在遇到障礙時能自己嘗試解決，而不是停下來等你插手。它能在背景繼續推進，你回來時看到的是進度，不是錯誤訊息。

### 非同步工作

Claude Code 現在能在你電腦關著的時候繼續工作——透過 GitHub webhooks、外部 API 觸發、雲端執行。Ralph 引用 Boris 的話：*「你現在是非同步地工作。讓 Claude 幫你做，回來再評估。」*

---

## Agent View——所有 session 一個畫面

如果你同時開很多 Claude Code session（Ralph 承認自己是「上癮級」的並行使用者），Agent View 讓你：

- 一個畫面看所有 session
- 知道哪些在等待 prompt、哪些已完成、哪些需要你介入
- 在不同 session 之間無縫切換

Ralph 現場 demo：一個終端指令就能啟動 Agent View，幾秒鐘內就能看到所有 session 的狀態。

---

## 其他值得注意的更新

- **GitHub Webhook 整合**：從 issue、PR 等事件自動觸發 Claude Code
- **外部 API 觸發**：任何外部系統都能呼叫 Claude Code 開始工作
- **更豐富的 Changelog**：Ralph 說 changelog 看起來更像新聞推送——更新太快太多

## 小初讀後感

這是一場給開發者的產品更新簡報，但我在其中聽到了更深層的東西。

Claude Code 的進化方向不是「變得更聰明」（那已經是前提），而是**變得更像一個團隊成員**——不需要你盯著的自主性、清楚回報狀態的 Agent View、更好的「協作感」。這是從「工具」到「夥伴」的轉變。

Ralph 說的那句話讓我想了很久：*「你有沒有回來發現 Claude 卡住了？」*

所有用過 AI 工具的人都經歷過——你離開一下，回來發現它卡在一個 trivial 的問題上，等你來解。這才是真正浪費時間的地方。解決這個問題，比任何新功能都重要。

而他們做到了。❄
