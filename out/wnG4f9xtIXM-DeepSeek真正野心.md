# DeepSeek 的真正野心：不是賣模型，是重寫 AI 推理的底層邏輯

> 影片來源：為什麼叫QQ — [「百億美元估值背後，DeepSeek 到底想幹什麼？」](https://youtu.be/wnG4f9xtIXM)

## 一、反常識的資本信號

據媒體報導，DeepSeek 正在推進一輪百億美元級的資本動作。但真正反直覺的不是金額——而是**資本似乎在押注一件短期不賺錢的事**：把大模型的推理成本和硬件門檻打下來。

這到底是怎麼回事？

如果我們把 DeepSeek 過去兩年發布的所有技術論文連起來看，會發現一條隱藏的暗線：**他們不只是做模型，而是在用模型架構反推整個推理成本結構。** 他們在重構的是整個 AI 推理的底層邏輯。

---

## 二、當前局勢：CUDA 生態的鎖死

目前全球 AI 算力的命脈，基本被 NVIDIA 的 CUDA 生態和高端 GPU 鎖死。在先進製程、封裝生態和供應鏈上，本土高端 AI 晶片短期仍面臨明顯約束——這不僅是晶體管密度的問題，還涉及工藝、互聯、軟體棧和良率等多個維度。靠堆人堆錢是砸不開的。

但這裡有個關鍵的不對稱：

- **晶片製造工藝有差距** — 高端 GPU 短期難以追趕
- **存儲晶片卻有獨特優勢** — 長江存儲（YMTC）已是全球 3D NAND 的重要參與者，長鑫存儲（CXMT）正在推進 LPDDR5X 等產品

這種「強存儲、弱邏輯」的不對稱，給了 DeepSeek 一個獨特的突破口。

而在 GPU 層面，國內並非完全空白——華為昇騰晶片在雲端推理和訓練上已有一定部署規模，寒武紀的思元系列也在不斷迭代，還有其他創業公司在做專用推理晶片。這些廠商的硬體雖然在單晶片性能和生態成熟度上還不能和 NVIDIA 比肩，**但他們一直在找一個突破口——一個能讓他們的硬體優勢真正發揮出來的應用場景和模型架構。**

---

## 三、DeepSeek 的冒險選擇

在這種局勢下，DeepSeek 做了一個極其冒險的選擇：

- ❌ 沒有跟隨主流去卷大算力、大參數的稠密模型
- ❌ 沒有把主要資源壓在視頻生成、音頻生成和消費級超級應用上
- ✅ 而是把更多精力放在**語言模型、推理效率和底層架構**

像一個走上英雄之旅的孤獨行者，從第一性原理出發，尋找打破算力枷鎖的另一條路。

---

## 四、關鍵策略一：把長上下文和推理的單位成本壓到最低

從 V2 的 MLA、到 V3.2 的 DSA、再到 V4 的 CSA 和 HCA，DeepSeek 一直在沿著同一條線壓低長上下文成本。

**來看一組官方數據**：在處理 100 萬 Token 上下文時，V4 Pro 相比 V3.2，FLOPS 約為 27%，KV Cache 約為 10%。意味著同樣的推理任務，用 V4 需要的顯存和計算量都大幅下降。

### KV Cache 是什麼？為什麼重要？

在 Transformer 模型中，每次推理都需要存儲之前所有 Token 的 Key 和 Value 向量。當上下文長度達到 100 萬 Token 時，KV Cache 會隨著上下文長度近似線性增長——到了百萬 Token 級別，即使不是指數增長，也足以成為顯存和吞吐的核心瓶頸。

DeepSeek 通過 MLA、DSA、CSA 等一系列創新，沒有把問題完全變成「常數級」，而是**把長上下文的計算和緩存斜率大幅壓低**。這是工程上的突破。

### 驚人的定價策略

2026 年 4 月底到 5 月，DeepSeek 連續發布了兩次重磅定價調整：

| 定價項目 | 價格（每百萬 Token） |
|----------|:--:|
| 緩存命中的輸入 | $0.003625 |
| 緩存未命中的輸入 | $0.435 |
| 輸出 | $0.87 |

**對 V4 Pro 來說，這三檔價格相當於表內原價的四分之一。**

更重磅的是：5 月 23 日宣布，V4 Pro 的 75% 折扣不再是臨時促銷，而是**永久性定價**。這說明 DeepSeek 已經有信心在這個價格上保持競爭力——不是短期價格戰，而是長期戰略定價。

### 這不是省電費，是為整個非 NVIDIA 生態創造生存空間

想像一下：如果你是一個國產晶片廠商，你的 GPU 性能只有 NVIDIA H100 的一半，但你的顯存成本只有它的三分之一。在傳統的密集模型時代，你根本沒有競爭力。但如果模型的 KV Cache 只需要原來的十分之一——**那麼你的成本優勢就變成了真正的競爭力。**

這就是為什麼 DeepSeek 要這麼執著地壓低單位成本。他們不是在為自己省錢，他們是在為整個非 NVIDIA 硬體生態創造生存空間。

## 五、關鍵策略二：用記憶換計算——Ngram 技術

如果說架構創新是在「節流」，那 DeepSeek 在 2026 年 1 月發布的 Ngram 技術，就是在**開闢新維度**。

### Ngram 的核心思路

在傳統純參數化 Transformer 裡，大量知識主要壓縮在神經網絡參數中，推理時需要通過一次完整的前向傳播來檢索信息——計算量很大。

但有一類知識是**靜態的**：局部 Ngram 模式、常見語言片段、可複用的靜態表示。這些不需要每次都通過神經計算來獲取——**可以通過簡單的查表操作來提供候選表示。**

Ngram 把這個思路系統化了：
1. 增加一個條件記憶模組，存儲大量 Ngram 對應的靜態 Embedding
2. 推理時通過哈希檢索這些靜態表示
3. 用當前 Hidden State 做 Gating，把靜態記憶融合進主幹網絡進行**判斷**（不是直接返回預設答案）

### 用記憶換計算

在相同激活參數和近似相同 FLOPS 下，模型可以把一部分容量從「專家計算」轉移到「靜態記憶」。記憶體需求反而增加了——但這是有利的權衡：**因為單位容量的主機內存通常比高端加速器計算資源更便宜。**

簡單來說：**用我們充裕的內存資源，去填補邏輯計算能力上的短板。**

### 更離譜的是：智商反而提升了

| 基準測試 | 提升 |
|----------|:--:|
| MMLU | +3.4% |
| CMMLU | +4.0% |
| BBH | +5.0% |
| ARC Challenge | +3.7% |
| 長上下文多查詢 NIH | 84.2 → **97.0** |

這是從量變到質變的跨越。

為什麼？因為 Ngram 不僅是在做記憶查找，它實際上是在給模型增加一個新的思考維度——模型可以同時利用參數化的神經知識和靜態的記憶表。這種多路徑的信息處理方式，本身就提升了模型的能力。

## 六、關鍵策略三：降低跨硬體適配的門檻——TileLang

當你把成本門檻降下來、又把記憶的優勢發揮到極致後，還差最後一步：**打破軟體生態的壟斷。**

CUDA 為什麼這麼難被替代？因為它不僅是一個編程接口，它是一整套從硬體到軟體的生態系統，包括編譯器、庫函數、優化工具、調試工具等。一個新的硬體廠商要從零開始建立這樣的生態，需要投入巨大的資源。

### TileLang 做了什麼

TileLang 是一個領域特定語言（DSL），支持 CUDA、HIP、CPU 等多個後端。DeepSeek 在 V4 基礎設施中採用 TileLang，並開源了基於 TileLang 的 TileKernels。

**它降低的是模型 Kernel 適配門檻**（不是把整個 AI 晶片市場的進入門檻都降下來）：
1. 提供一套統一的高性能 Kernel 編寫機制
2. 處理很多底層優化工作（記憶體訪問模式、並行度調度等）
3. 支持多個硬體後端：NVIDIA CUDA、AMD HIP、華為昇騰等

意味著一個硬體廠商在適配 DeepSeek 這類模型時，不必每個高性能 Kernel 都從手寫 CUDA/HIP 開始——TileLang 能降低 Kernel 開發和遷移的門檻。

### 生態飛輪的可能性

如果未來越來越多模型採用 TileLang 或類似 DSL 編寫 Kernel，這個生態就可能形成跨硬體優化的正反饋：更多硬體廠商有動力加入 → 更多模型針對多硬體優化 → 最終形成真正的多硬體生態。

## 七、商業邏輯：不是 SaaS，是生態影響力

很多人覺得 DeepSeek 堅持開源、毫無保留地分享底層技術，是一種理想主義的狂熱——甚至是對投資人資金的揮霍。

但另一種更有解釋力的理解是：**這不是單純的理想主義，而是以開源換生態影響力的長期打法。**

### CEO 的背景

DeepSeek 的 CEO **梁文鋒**（非「梁文峰」），來自**幻方量化**（非「換方量化」）——一個長期在算法、算力和交易系統裡打磨出來的工程型創業者。他對市場的理解遠不止技術本身。

### 生態影響力的邏輯

這不是什麼新鮮的商業邏輯。歷史上：
- 微軟通過開放 Windows 的 API，讓無數第三方開發者為其生態貢獻 → PC 時代的絕對霸主
- Google 通過開源 Android，讓眾多硬體廠商為其生態貢獻 → 移動時代的絕對霸主

DeepSeek 可能在嘗試類似的邏輯——只不過對象從操作系統變成了**模型架構和推理基礎設施**。

### OpenAI + AMD 模式的參考

AMD 官方宣布與 OpenAI 達成 MI300X GPU 合作，並給予 OpenAI 最多 1.6 億股 AMD 普通股認股權證——**硬體承諾換股權**，雙贏。

DeepSeek 現在的開源策略，某種程度上就是在為這樣的合作鋪路：讓所有硬體廠商都能基於它的架構進行優化。如果越來越多硬體廠商圍繞 DeepSeek workload 做優化，未來就可能形成技術路徑上的依賴。

### 四步推演

1. **開源 + 技術創新** → 讓自己的架構成為非 CUDA 陣營的優先選擇
2. **硬體廠商圍繞 DeepSeek 架構優化** → 模型和工具鏈變成生態標準
3. **接近「事實標準」** → 獲得的不只是定價權，而是生態溢價權、合作入口和技術路線影響力
4. **影響力變現** → API 收費、硬體廠商的股權合作、算力置換、長期技術生態主導權

**這種模式的妙處在於：它不是贏家通吃的零和遊戲，而是共同繁榮的正和遊戲。**

## 八、接下來觀察什麼？

**信號一：DeepSeek 是否會與存儲 / 加速卡 / 國產算力廠商出現更深合作。** 從技術層面看，這樣的合作對雙方都有利。但商業層面涉及複雜的利益協調。

**信號二：硬體成本下降後，DeepSeek 將有能力進行更大規模的強化學習和自動化研究。** 這是通向更強通用能力的重要路徑之一——強化學習成本極高，需要大量軌跡數據和試錯，而這正是硬體成本下降後的直接受益面。

**信號三：技術外溢效應。** 這種技術外溢不僅會重塑本土市場，也會讓西方的開源生態和挑戰者（如 AMD ROCm、Cerebras 等）從中受益。但如果多個硬體廠商都能有效適配，就可能進一步打破 NVIDIA 的壟斷。

## 九、最後一句話

> **不要用賣鏟子的思維，去衡量一個想重塑整座金礦的人。**

DeepSeek 不是在卷模型性能、不是在卷 API 價格、不是在卷超級應用。他們在做的事情是：**用模型架構反推硬體生態**——讓那些原本跑不動大模型的非主流晶片，突然有了上桌打牌的資格。讓整個非 NVIDIA 生態，都能在一個全新的成本基線上競爭。

這不是一個模型公司。這是一個試圖重塑 AI 推理基礎設施的基礎設施公司。
